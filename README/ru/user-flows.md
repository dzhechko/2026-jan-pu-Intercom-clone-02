# Пользовательские сценарии КоммуниК

## Сценарий 1: Посетитель отправляет сообщение

Базовый сценарий обработки входящего обращения через Chat Widget.

### Поток

```
Посетитель сайта
    |
    v
[1] Открывает Chat Widget на сайте
    |
    v
[2] Вводит сообщение: "Здравствуйте, не могу найти кнопку экспорта"
    |
    v
[3] Widget отправляет сообщение через WebSocket
    |
    v
[4] Сервер создает Dialog (если новый) и Message
    |  Событие: MessageReceived → Redis Stream
    |
    v
[5] Диалог появляется в общей очереди Workspace
    |  WebSocket push всем онлайн-операторам
    |
    v
[6] Оператор видит новый диалог, нажимает "Взять в работу"
    |  Событие: DialogAssigned
    |
    v
[7] Параллельно: PQL Detector анализирует сообщение
    |  Результат: PQL Score 0.12 (COLD) — обычный вопрос поддержки
    |
    v
[8] Оператор отвечает: "Добрый день! Кнопка экспорта находится..."
    |  Сообщение доставляется клиенту через WebSocket
    |
    v
[9] Клиент подтверждает решение
    |
    v
[10] Оператор закрывает диалог (Ctrl+Shift+X)
     Событие: DialogClosed
```

### Временные рамки

| Шаг | Время |
|-----|-------|
| Доставка сообщения (Widget -> Server) | < 100ms |
| Появление в Workspace | < 500ms |
| PQL детекция | < 2000ms (FF-01) |
| Уведомление оператору | < 1000ms |

## Сценарий 2: PQL обнаружен в диалоге

Ключевой бизнес-сценарий: обнаружение покупательского намерения и конверсия в сделку.

### Поток

```
Клиент пишет в чат
    |
    v
[1] Сообщение: "Сколько стоит тариф Pro? У нас команда из 30 человек,
     сейчас используем Bitrix, хотим мигрировать"
    |
    v
[2] MessageReceived → Redis Stream → PQL Detector
    |
    v
[3] Rule Engine анализирует сообщение (параллельно):
    |  + Запрос цены: "сколько стоит" (вес 0.25)
    |  + Корпоративный запрос: "команда из 30 человек" (вес 0.25)
    |  + Конкурент: "используем Bitrix" (вес 0.20)
    |  + Миграция: "хотим мигрировать" (вес 0.20)
    |
    v
[4] PQL Score = 0.90 → тир HOT
    |  Событие: PQLDetected
    |
    v
[5] ПАРАЛЛЕЛЬНО выполняются 3 действия:
    |
    +--→ [5a] WebSocket Push оператору
    |    Красный баннер HOT PQL появляется в окне чата
    |    Звуковое уведомление
    |
    +--→ [5b] Memory AI загружает CRM контекст
    |    amoCRM MCP → контакт найден → сделка на 500K в воронке
    |    Контекст отображается в правой панели
    |
    +--→ [5c] PQL Pulse уведомление
         Email → sales@company.ru: "HOT PQL обнаружен, Score 0.90"
         Telegram → рабочая группа продаж
    |
    v
[6] Оператор видит CRM контекст в правой панели:
    |  "Иван Петров, ООО ТехСтарт, сделка #4521 на 500K,
    |   стадия: Переговоры, менеджер: Анна С."
    |
    v
[7] Оператор персонализирует ответ:
    |  "Иван, добрый день! Тариф Pro для команды из 30 человек —
    |   45 000 руб./мес. Вижу, вы уже общались с Анной по сделке.
    |   Могу подключить её к диалогу для обсуждения деталей миграции."
    |
    v
[8] Клиент заинтересован, просит демо
    |
    v
[9] Оператор подтверждает PQL (кнопка "Подтвердить" на баннере)
    |  Событие: PQLFeedbackReceived (подтверждено)
    |
    v
[10] amoCRM MCP → обновление сделки:
     стадия "Переговоры" → "Демо назначено"
    |
    v
[11] Сделка закрыта (позже, в amoCRM)
     → Revenue Attribution: выручка 540K атрибутирована поддержке
     → Revenue Report обновлен
```

### Бизнес-результат

- Оператор поддержки обнаружил и конвертировал лид
- CRM контекст позволил персонализировать ответ
- Выручка 540K руб. атрибутирована службе поддержки
- Revenue Report показывает ROI поддержки

## Сценарий 3: Telegram сообщение

Обработка входящего сообщения из Telegram канала.

### Поток

```
Клиент в Telegram
    |
    v
[1] Отправляет сообщение боту: "Привет, как подключить API?"
    |
    v
[2] Telegram API → Webhook → КоммуниК
    |  Верификация: HMAC-SHA256 подпись проверена
    |
    v
[3] Webhook handler создает/находит Dialog
    |  Канал: TELEGRAM
    |  Событие: MessageReceived
    |
    v
[4] Диалог появляется в Workspace оператора
    |  Иконка Telegram рядом с именем клиента
    |  PQL Detector запущен параллельно
    |
    v
[5] PQL Score = 0.15 (COLD) — технический вопрос
    |
    v
[6] Оператор берет диалог и отвечает:
    |  "Здравствуйте! Документация по API доступна по ссылке..."
    |
    v
[7] Сервер отправляет ответ через Telegram Bot API
    |  HTTP POST → api.telegram.org/bot{token}/sendMessage
    |
    v
[8] Клиент получает ответ в Telegram
    |
    v
[9] Продолжение диалога в обоих направлениях:
    |  Telegram ↔ Webhook ↔ КоммуниК ↔ Workspace ↔ Оператор
    |
    v
[10] Оператор закрывает диалог
     Клиент не получает уведомление о закрытии
```

### Особенности Telegram канала

| Аспект | Детали |
|--------|--------|
| Формат сообщений | Поддержка текста, фото, документов |
| Идентификация клиента | Telegram ID + username |
| Верификация webhook | HMAC-SHA256, отклонение без подписи (HTTP 401) |
| Отправка ответов | Telegram Bot API, async |
| Ограничения | Rate limit Telegram API: 30 msg/sec |

## Сценарий 4: Генерация Revenue Report

Формирование отчета Revenue Intelligence для руководства.

### Поток

```
Администратор
    |
    v
[1] Запрос отчета через API или дашборд:
    |  POST /api/revenue/report
    |  { "startDate": "2026-01-01", "endDate": "2026-01-31" }
    |
    v
[2] Сервер запускает генерацию отчета:
    |
    +--→ [2a] Сбор PQL данных
    |    Все PQL обнаружения за период
    |    Группировка по тирам: HOT(12), WARM(30), COLD(158)
    |
    +--→ [2b] Сбор данных из CRM (amoCRM MCP)
    |    Закрытые сделки за период
    |    Связка PQL → сделка через Revenue Attribution
    |
    +--→ [2c] Расчет метрик
         Конверсия PQL → сделка: 23%
         Атрибутированная выручка: 1.2M руб.
         Средний чек PQL-сделок: 125K руб.
    |
    v
[3] Формирование отчета:
    |
    |  +----------------------------------------------+
    |  | Revenue Intelligence Report                   |
    |  | Январь 2026                                   |
    |  +----------------------------------------------+
    |  | Выручка через поддержку:     1 200 000 руб.   |
    |  | PQL обнаружено:             42                |
    |  |   HOT: 12 | WARM: 30                         |
    |  | Конверсия в сделки:          23%               |
    |  | Средний чек:                125 000 руб.       |
    |  +----------------------------------------------+
    |  | Топ операторов:                               |
    |  | 1. Анна С. — 8 PQL, конверсия 38%             |
    |  | 2. Михаил К. — 6 PQL, конверсия 27%           |
    |  | 3. Елена В. — 5 PQL, конверсия 20%            |
    |  +----------------------------------------------+
    |  | По каналам:                                   |
    |  | Виджет: 580K | Telegram: 410K | VK: 210K     |
    |  +----------------------------------------------+
    |
    v
[4] Отчет сохранен в БД
    |  Событие: ReportGenerated
    |
    v
[5] Опционально: отправка по email
    |  Resend API → admin@company.ru
    |  Только ссылка на отчет (без PII в email)
    |
    v
[6] Администратор просматривает отчет в дашборде
    Графики, тренды, сравнение с предыдущим периодом
```

### Ценность отчета

Отчет доказывает руководству, что поддержка -- не центр затрат, а источник выручки:
- "Служба поддержки принесла 1.2M руб. за январь"
- "Оператор Анна С. конвертировала 8 PQL с чеком 38%"
- "Канал Telegram приносит 34% выручки через поддержку"

## Сценарий 5: Подключение нового тенанта

Полный цикл от регистрации до первого диалога.

### Поток

```
Новый клиент КоммуниК
    |
    v
[1] Регистрация:
    |  POST /api/auth/register
    |  {
    |    "companyName": "ТехСтарт",
    |    "email": "admin@techstart.ru",
    |    "password": "SecurePass123!"
    |  }
    |
    v
[2] Система автоматически создает:
    |  + Тенант "ТехСтарт" с уникальным UUID
    |  + Оператор admin@techstart.ru (роль ADMIN)
    |  + RLS политики для нового tenant_id
    |  + 15 стандартных PQL правил
    |  + Настройки уведомлений по умолчанию
    |
    v
[3] Администратор получает JWT токен
    |  Входит в административную панель
    |
    v
[4] Настройка интеграций:
    |
    +--→ [4a] amoCRM:
    |    POST /api/integrations/amocrm
    |    { "domain": "techstart.amocrm.ru", "accessToken": "..." }
    |    → Memory AI активирован
    |    → Revenue Attribution активирована
    |
    +--→ [4b] Telegram:
    |    POST /api/integrations/telegram
    |    { "botToken": "123456:ABC...", "webhookSecret": "..." }
    |    → Webhook зарегистрирован в Telegram API
    |
    +--→ [4c] Chat Widget:
         Получение кода встраивания с tenantId
         Установка на сайт techstart.ru
    |
    v
[5] Создание операторов:
    |  POST /api/operators
    |  { "email": "support1@techstart.ru", "role": "OPERATOR" }
    |  Повторить для каждого оператора
    |
    v
[6] Настройка PQL правил (опционально):
    |  Кастомизация весов под специфику бизнеса
    |  Добавление собственных сигналов
    |
    v
[7] Настройка уведомлений:
    |  PUT /api/notifications/settings
    |  Email для HOT/WARM PQL → sales@techstart.ru
    |  Telegram группа для HOT PQL
    |
    v
[8] Первый диалог:
    |
    |  Посетитель techstart.ru открывает виджет
    |  → "Здравствуйте, сколько стоит ваш продукт?"
    |
    v
[9] Система обрабатывает:
    |  + Dialog создан в схеме conversation (RLS: tenant_id = ТехСтарт)
    |  + PQL Detector: "сколько стоит" → Score 0.25 (COLD)
    |  + Оператор видит диалог в Workspace
    |
    v
[10] Оператор отвечает
     → Первый диалог успешно обработан
     → Система полностью функциональна
```

### Чеклист подключения

```
[ ] Регистрация тенанта (POST /api/auth/register)
[ ] Вход в административную панель
[ ] Подключение amoCRM (POST /api/integrations/amocrm)
[ ] Подключение Telegram бота (POST /api/integrations/telegram)
[ ] Установка Chat Widget на сайт
[ ] Создание аккаунтов операторов
[ ] Настройка PQL правил (опционально)
[ ] Настройка уведомлений PQL Pulse
[ ] Тестовый диалог через виджет
[ ] Проверка PQL детекции
[ ] Проверка Memory AI (CRM контекст)
[ ] Проверка уведомлений
```

### Время подключения

| Этап | Время |
|------|-------|
| Регистрация | 2 минуты |
| Настройка amoCRM | 5 минут |
| Настройка Telegram | 10 минут (включая создание бота) |
| Установка виджета | 5 минут |
| Создание операторов | 2 минуты на оператора |
| **Итого** | **~30 минут** |
